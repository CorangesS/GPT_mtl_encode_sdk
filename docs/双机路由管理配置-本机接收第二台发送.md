# 双机路由管理配置：本机接收、第二台发送 build/yuv，并在 Easy-NMOS 中查看与操作

本文与 [双机路由管理配置-本机发送第二台接收.md](双机路由管理配置-本机发送第二台接收.md) **角色对调**：**本机**运行 Easy-NMOS 与**自研接收**（Receiver + 收流），**第二台电脑**运行**自研发送**（Sender + `st2110_send`）。发送端使用 **enp4s0 / 192.168.1.110**，接收端使用 **enp6s0 / 192.168.1.100**；发送命令显式指定 **30 fps**；**发送端与接收端均需启动 IS-05**（示例使用 `app4.py`）。

---

## 一、角色与地址约定


| 机器        | 网卡与 IP                       | 角色                                                                                                                     |
| --------- | ---------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **本机**    | **enp6s0**，**192.168.1.100** | ① 运行 **Easy-NMOS**（Registry + admin）② 注册 **Receiver**、运行 **IS-05**、运行 **st2110_record** 或 **is05_receiver_daemon** 收组播 |
| **第二台电脑** | **enp4s0**，**192.168.1.110** | 注册 **Sender**、运行 **IS-05**、用 **st2110_send**（**--fps 30**）向组播发 **build/yuv**                                           |


要求：两机互通；能访问 Registry（下文以 `http://192.168.1.200` 为例，按你实际 Easy-NMOS 地址修改）。**收发使用同一组播地址与端口**（仍以 **239.0.0.1:5004** 为例）。`ip -br addr` 可核对本机 **enp6s0** 是否为 **192.168.1.100**。

## 二、本机（192.168.1.100，enp6s0）配置

### 2.1 确认 Easy-NMOS 已运行

- 浏览器能打开 Controller，例如 `http://192.168.1.200/admin`（以实际为准）。
- 本机防火墙放行 Registry/HTTP 端口。

### 2.2 本机网卡 IP（收流用 enp6s0）

若尚未配置：

```bash
sudo ip addr add 192.168.1.100/24 dev enp6s0
```

（接口名以 `ip link show` / `ip -br addr` 为准。）

### 2.3 本机注册为「接收端」（admin 的 Receivers 里可见）

在本机项目目录执行，`--href` 为本机 **IS-05 可达地址**（本机 IP + 端口）：

```bash
cd /path/to/GPT_mtl_encode_sdk
export REGISTRY_URL=http://192.168.1.200

python3 routing/scripts/register_node_example.py --mode receiver \
  --heartbeat --interval 1 \
  --href http://192.168.1.100:9090/ \
  --save-config .nmos_node.json
```

- 若 9090 被占用，可改为 9091，并同步修改下文 IS-05 端口与 `--href`。
- 注册脚本建议**常驻**，维持心跳。

### 2.4 本机运行 IS-05 服务

在**本机**另开终端（与原文档「仅第二台起 IS-05」不同，**本机作为接收端也必须起 IS-05**，以便 admin / NMOS 对 Receiver 做 Connect）：

```bash
cd /path/to/GPT_mtl_encode_sdk
python3 routing/is05_server/app4.py
```

默认监听 **9090**；若改端口，使用环境变量或 `app4.py` 说明中的端口配置，并与 `--href` 一致。防火墙放行对应端口。

### 2.5 本机实际收流

**方式 A：直接 st2110_record（与第二台发送参数一致）**

先在本机启动接收，再在第二台启动发送：

```bash
cd /path/to/GPT_mtl_encode_sdk/build

./st2110_record --ip 239.0.0.1 --video-port 5004 --audio-port 0 \
  --width 1920 --height 1080 --max-frames 3000 recv.mp4 \
  --port kernel:enp6s0 --sip 192.168.1.100 --no-ptp
```

**方式 B：admin Connect + is05_receiver_daemon（或 is05_receiver_only_daemon）**

1. 本机运行注册脚本、`app4.py`，再运行收流 daemon（参数按你仓库说明）。
2. 在 admin 中对本机 **Receiver** 执行 **Connect**，选择第二台注册的 **Sender**；SDP 中的组播/端口应与第二台 `st2110_send` 一致（239.0.0.1、5004）。

## 三、第二台电脑（192.168.1.110，enp4s0）配置

### 3.1 网络与项目准备

- 第二台能 **ping 通 本机 192.168.1.100** 与 Registry 地址。
- 安装 Python 依赖：`pip install flask requests`（按需）。

### 3.2 第二台网卡 IP（发流用 enp4s0）

```bash
sudo ip addr add 192.168.1.110/24 dev enp4s0
```

### 3.3 第二台注册为「发送端」（admin 的 Senders 里可见）

在**第二台**执行：

```bash
cd /path/to/GPT_mtl_encode_sdk
export REGISTRY_URL=http://192.168.1.200

python3 routing/scripts/register_node_example.py --mode sender \
  --heartbeat --interval 1 \
  --href http://192.168.1.110:9090/ \
  --save-config .nmos_node.json
```

`--href` 必须为**第二台 IP + IS-05 端口**，以便 Controller 拉 **transportfile**、发 **PATCH** 到第二台。

### 3.4 第二台运行 IS-05 服务（与接收场景对称）

在**第二台**另开终端，**与接收端相同命令**：

```bash
cd /path/to/GPT_mtl_encode_sdk
python3 routing/is05_server/app4.py
```

第二台防火墙放行 **9090**（或你选用的端口）。

### 3.5 第二台实际发送 build/yuv（30 fps）

在**第二台**、**本机已先启动接收**（或已 Connect 且 daemon 就绪）后执行：

```bash
cd /path/to/GPT_mtl_encode_sdk/build

./st2110_send --url yuv420p10le_1080p.yuv --width 1920 --height 1080 \
  --fps 30 \
  --duration 60 --audio-port 0 \
  --ip 239.0.0.1 --video-port 5004 \
  --port kernel:enp4s0 --sip 192.168.1.110 --no-ptp
```

说明：

- `--fps 30`：发送节奏与 SDP 中的帧率信息按 **30 fps** 配置（`st2110_send` 支持 `--fps`；若不指定则默认约 59.94）。
- `--port kernel:enp4s0`、`--sip 192.168.1.110`：与本文「发送端 enp4s0 + 192.168.1.110」一致。
- `--ip` / `--video-port` 必须与本机 **st2110_record** 或 **connection_state** 中组播一致。

## 四、在 Easy-NMOS 中查看与操作

- 打开 admin（例如 `http://192.168.1.200/admin`）。
- **Senders**：应能看到第二台的「Video Sender (ST2110)」。
- **Receivers**：应能看到本机的「Video Receiver (ST2110)」。
- 对 **Receiver** 执行 **Connect**，选择第二台的 **Sender**。

命令行断开 Receiver 时，将 `RECEIVER_NODE` 设为本机 IS-05，例如：

```bash
export RECEIVER_NODE="http://192.168.1.100:9090"
export RECEIVER_ID="<本机 Query 中 receiver 的 id>"
```

具体 PATCH 示例见原文档 [§4.5 不通过 admin 的命令行断开](双机路由管理配置-本机发送第二台接收.md)。

## 五、操作顺序速查（第二台发、本机收）


| 步骤  | 位置  | 操作                                                                        |
| --- | --- | ------------------------------------------------------------------------- |
| 1   | 本机  | Easy-NMOS 可访问；`enp6s0` 配 **192.168.1.100/24**                             |
| 2   | 本机  | `--mode receiver`，`--href http://192.168.1.100:9090/`，心跳常驻                |
| 3   | 本机  | `python3 routing/is05_server/app4.py`                                     |
| 4   | 第二台 | `enp4s0` 配 **192.168.1.110/24**                                           |
| 5   | 第二台 | `--mode sender`，`--href http://192.168.1.110:9090/`，心跳常驻                  |
| 6   | 第二台 | `python3 routing/is05_server/app4.py`                                     |
| 7   | 本机  | 先启 **st2110_record**（或 daemon），`--port kernel:enp6s0 --sip 192.168.1.100` |
| 8   | 第二台 | 再启 **st2110_send**，`--fps 30`，`--port kernel:enp4s0 --sip 192.168.1.110`  |
| 9   | 任意  | admin 中 Connect / 或命令行 IS-05 与原文档一致                                       |

## 六、参数对照（第二台发送与本机接收须一致）


| 参数      | 第二台 st2110_send            | 本机 st2110_record           |
| ------- | -------------------------- | -------------------------- |
| 组播 IP   | --ip 239.0.0.1             | --ip 239.0.0.1             |
| 视频端口    | --video-port 5004          | --video-port 5004          |
| 分辨率     | --width 1920 --height 1080 | --width 1920 --height 1080 |
| 帧率      | **--fps 30**               | 与发送一致（收端按流内 SDP/实际码流）      |
| 网卡      | --port kernel:**enp4s0**   | --port kernel:**enp6s0**   |
| 源/本机 IP | --sip **192.168.1.110**    | --sip **192.168.1.100**    |

## 七、常见问题

- **admin 看不到 Sender/Receiver**：检查两台注册脚本、`REGISTRY_URL`、心跳、防火墙。
- **Connect 失败**：确认两台 `**app4.py` 均在运行**，`--href` 与浏览器可达 IP:端口一致。
- **本机收不到流**：确认组播 IP/端口一致、先收后发、`enp6s0` 已加入 192.168.1.0/24；必要时抓包 239.0.0.1:5004。
