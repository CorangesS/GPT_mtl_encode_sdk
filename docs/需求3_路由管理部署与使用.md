# 需求3：路由管理 — 部署与使用

对应 **需求.md**：AMWA NMOS、集中注册/发现/调度/路由与监控、可视化界面、自研收发端与外购设备的发现与连接管理。本文说明**如何运行当前项目中的路由管理部分**，以及**如何在双机上完成需求内容的测试**。

---

## 一、路由管理组成

- **Registry + Controller**：由 **Easy-NMOS** 提供（一键部署），提供 IS-04 注册/发现与 IS-05 连接管理，以及可视化 admin 界面（`http://<Easy-NMOS-IP>/admin`）。
- **本仓库路由部分**：`routing/` 下注册脚本与 IS-05 服务端，用于把**自研节点**（发送端/接收端）注册到同一 Registry，并在 admin 里对自研 Receiver 做 Connect 时驱动 MTL 收流。

---

## 二、单机/双机角色约定

- **机器 A（本机）**：运行 **Easy-NMOS**（例如 192.168.1.200），提供 Registry 与 admin；可选在本机再运行自研节点（注册脚本 + IS-05）。
- **机器 B（第二台）**：可选仅用浏览器打开 admin；或运行**自研节点**（注册脚本 + IS-05 + 可选收流 daemon），在 admin 里作为可连接的 Receiver/Sender 出现。

两机需网络互通（能访问 Easy-NMOS 的 IP；若 B 跑自研节点，A 或操作 admin 的 PC 需能访问 B 的 IS-05 端口）。

---

## 三、本机（机器 A）需要做的

### 3.1 必须

1. **启动并保持 Easy-NMOS 运行**，确保 admin 可访问：`http://192.168.1.200/admin`（IP 换成实际）。
2. **防火墙**：放行 80 端口，以便机器 B 或其它 PC 能打开 admin。

### 3.2 可选（自研节点也跑在 A 上）

若在 A 上运行自研接收/发送节点：

- 运行注册脚本：`REGISTRY_URL=http://192.168.1.200`，`--href http://192.168.1.200:9090/`（或本机实际 IP），`--save-config .nmos_node.json`（接收端时）。
- 运行 IS-05 服务：`python3 routing/is05_server/app.py`（端口与 `--href` 一致，默认 9090；若占用可 `IS05_PORT=9091` 并相应改 `--href`）。
- 可选：运行 `is05_receiver_daemon` 或 `st2110_record`，使「在 admin 里点连接」真正收流。

---

## 四、第二台电脑（机器 B）需要做的

### 4.1 仅用 admin 做路由管理

- 浏览器打开 `http://192.168.1.200/admin`，无需安装本仓库。

### 4.2 在 B 上运行自研节点（在 admin 里可见、可 Connect）

假设 B 的 IP 为 192.168.1.201。

1. **安装**：在 B 上克隆本仓库，安装 Python 与依赖（如 `pip install flask requests`）。

2. **注册节点**（向 A 的 Registry 注册，并声明 B 的 IS-05 地址）：
   ```bash
   cd /path/to/GPT_mtl_encode_sdk
   export REGISTRY_URL=http://192.168.1.200

   # 接收端（在 admin 的 Receivers 里可见，且可 Connect）
   python3 routing/scripts/register_node_example.py --mode receiver \
     --heartbeat --interval 30 \
     --href http://192.168.1.201:9090/ \
     --save-config .nmos_node.json
   ```
   - `--href` 必须是 **B 的 IP + IS-05 端口**，这样在任意 PC 打开 admin 做 Connect 时，请求会发到 B。
   - 若 9090 被占用：`IS05_PORT=9091 python3 routing/is05_server/app.py`，并改为 `--href http://192.168.1.201:9091/`。

3. **运行 IS-05 服务**（在 B 上）：
   ```bash
   python3 routing/is05_server/app.py
   ```
   端口与 `--href` 一致。

4. **防火墙**：B 放行 IS-05 端口（9090 或 9091），以便 admin 所在浏览器能访问。

5. **（可选）真正收流**：在 B 上运行 `./build/is05_receiver_daemon`，或根据 `connection_state.json` 启动 `st2110_record`。

**发送端**：若要在 admin 的 Senders 里看到自研发送端，在 B（或 A）上执行：
```bash
python3 routing/scripts/register_node_example.py --mode sender \
  --heartbeat --interval 30 --href http://192.168.1.201:9090/
```
**收发一体**：`--mode both`，并同样运行 IS-05 服务（接收端要能 Connect）。

## 五、双机完成需求测试的流程

1. **机器 A**：启动 Easy-NMOS，确认 `http://192.168.1.200/admin` 可打开。
2. **机器 B**：按 §4.2 运行注册脚本（`--mode receiver` 或 `both`）与 IS-05 服务；可选运行 is05_receiver_daemon。
3. **在 A 或 B 的浏览器**打开 `http://192.168.1.200/admin`，在 **Nodes / Receivers / Senders** 中应看到 B 注册的自研节点。
4. **发现**：在 admin 查看所有 Node、Device、Sender、Receiver。
5. **连接（路由）**：选一个 Receiver（如 B 注册的「Video Receiver (ST2110）」），执行 Connect，选择某 Sender（如 Easy-NMOS 自带的虚拟 Sender 或 B 注册的 Sender）；确认/激活。
6. 若 B 上运行了 IS-05 与 daemon，连接后 B 会按 connection_state 收流并写文件；若仅做「发现与连接」验证，可不跑 daemon。

外购 ST2110 设备：将其 Registry 配置为 `http://192.168.1.200`，即可在 admin 中与自研节点一起发现、连接与管理。

---

## 六、自研 Receiver 无法 Connect/Active/Staged 时

- **原因**：admin 发出的 IS-05 请求会发到该 Node 的 `--href`；若 href 不可达（如填了 127.0.0.1 且从别处打开 admin），或未运行 IS-05 服务，则无法操作。
- **处理**：① 注册时 `--href` 填**运行 IS-05 的那台机器的 IP + 端口**，且从打开 admin 的浏览器要能访问；② 在该机器上运行 `routing/is05_server/app.py`；③ 接收端注册时加 `--save-config .nmos_node.json`，IS-05 服务从同目录读取 receiver_id。

---

## 七、命令速查

| 目的 | 命令 |
|------|------|
| 仅接收端注册 | `python3 routing/scripts/register_node_example.py --mode receiver --heartbeat --interval 30 --href http://<本机IP>:9090/ --save-config .nmos_node.json` |
| 仅发送端注册 | `python3 routing/scripts/register_node_example.py --mode sender --heartbeat --interval 30 --href http://<本机IP>:9090/` |
| 收发一体注册 | `python3 routing/scripts/register_node_example.py --mode both ... --save-config .nmos_node.json` |
| 启动 IS-05 服务 | `python3 routing/is05_server/app.py`（或 `IS05_PORT=9091 ...`） |
| 打开路由管理界面 | 浏览器访问 `http://<Easy-NMOS-IP>/admin` |

将 `<本机IP>` 换为运行注册脚本与 IS-05 服务的机器 IP；`REGISTRY_URL` 设为 Easy-NMOS 的地址。
