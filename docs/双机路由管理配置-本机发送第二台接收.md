# 双机路由管理配置：本机发送 build/yuv、第二台接收，并在 Easy-NMOS 中查看与操作

本文说明在**本机已运行 Easy-NMOS** 的前提下，如何配置**本机发送**、**第二台电脑接收**，并在 Easy-NMOS 的 admin 界面中**查看与操作**（发现节点、执行连接等）。本机使用项目中的 **build/yuv** 文件作为发送源。

---

## 一、角色与地址约定

| 机器 | 假设 IP | 角色 |
|------|---------|------|
| **本机** | 192.168.1.100 | ① 运行 **Easy-NMOS**（Registry + admin） ② 运行**自研发送**：注册为 Sender、用 `st2110_send` 发送 **build/yuv** |
| **第二台电脑** | 192.168.1.110 | 运行**自研接收**：向本机 Registry 注册 Receiver、运行 IS-05 服务、运行收流程序接收本机发出的组播流 |

要求：两机**网络互通**（同网段或路由可达），第二台能访问 `http://192.168.1.100`，本机或第二台能访问第二台的 IS-05 端口（如 9090）。**收发使用同一组播地址与端口**（下文以 239.0.0.1:5004 为例）。

---

## 二、本机（192.168.1.100）配置

### 2.1 确认 Easy-NMOS 已运行

- 浏览器打开 **http://192.168.1.100/admin**，能打开 Controller 界面即可。
- 本机防火墙放行 **80** 端口。

### 2.2 本机注册为「发送端」（在 admin 的 Senders 里可见）

在本机项目目录下执行，使 admin 里能看到「自研发送节点」：

```bash
cd /path/to/GPT_mtl_encode_sdk
export REGISTRY_URL=http://192.168.1.100

python3 routing/scripts/register_node_example.py --mode sender \
  --heartbeat --interval 30 \
  --href http://192.168.1.100:9090/
```

- 若 9090 已被占用，可换端口，例如 `--href http://192.168.1.100:9091/`（本机若需对自研 Sender 做 IS-05 则也需在本机起 IS-05 并监听 9091；当前仅「发现」可暂不启动）。
- 保持该脚本**常驻**（或后台运行），以便心跳维持注册；否则一段时间后 Registry 会剔除该节点。

注册成功后，在 **http://192.168.1.100/admin** 的 **Nodes / Senders** 中应能看到本机的「Video Sender (ST2110)」。

### 2.3 本机配置直连网卡与 IP（用于实际发流）

本机需要用一块网卡向第二台发组播。假设直连网卡为 **enp4s0**（以 `ip link show` 实际为准）：

```bash
# 临时配置 IP（与第二台同网段）
sudo ip addr add 192.168.1.100/24 dev enp4s0
```

若本机 192.168.1.100 已在其他网卡上（如 Easy-NMOS 所在网卡），则发流网卡可用同网段其他 IP，例如 192.168.1.101，只要与第二台 192.168.1.110 能互通即可。下例统一用 **192.168.1.100** 表示本机发流网卡 IP（你按实际修改）。

### 2.4 本机实际发送 build/yuv（st2110_send）

在**本机**运行发送程序，将 **build** 目录下的 YUV 文件以 ST2110 组播发出。组播地址与端口需与第二台接收端一致（例如 **239.0.0.1、5004**）：

```bash
cd /path/to/GPT_mtl_encode_sdk/build

./st2110_send --url yuv420p10le_1080p.yuv --width 1920 --height 1080 \
  --duration 60 --audio-port 0 \
  --ip 239.0.0.1 --video-port 5004 \
  --port kernel:enp4s0 --sip 192.168.1.100 --no-ptp
```

- `--url`：YUV 文件路径；在 build 目录下执行即 `yuv420p10le_1080p.yuv`，在项目根目录则用 `build/yuv420p10le_1080p.yuv`。
- `--port kernel:enp4s0`：使用内核网卡发流，与上面配置 IP 的网卡一致。
- `--sip`：本机该网卡 IP。
- `--ip` / `--video-port`：组播地址与端口，**必须与第二台 st2110_record 或 daemon 使用的参数一致**。

发送启动后，本机即在 239.0.0.1:5004 上发 ST2110 视频流。

## 三、第二台电脑（192.168.1.110）配置

### 3.1 网络与项目准备

- 第二台能 **ping 通 192.168.1.100**，并能访问 **http://192.168.1.100**。
- 在第二台克隆/拷贝本仓库，安装 Python 与依赖：`pip install flask requests`。
- 第二台需有一块网卡与本机同网段（或路由可达），用于收组播；为该网卡配置 IP，例如：
  ```bash
  sudo ip addr add 192.168.1.110/24 dev enp6s0
  ```
  （接口名以 `ip link show` 为准。）

### 3.2 第二台注册为「接收端」（在 admin 的 Receivers 里可见、可 Connect）

在**第二台**上执行，向本机 Registry 注册接收节点，并声明 IS-05 地址为本机可访问的 192.168.1.110:9090：

```bash
cd /path/to/GPT_mtl_encode_sdk
export REGISTRY_URL=http://192.168.1.100

python3 routing/scripts/register_node_example.py --mode receiver \
  --heartbeat --interval 30 \
  --href http://192.168.1.110:9090/ \
  --save-config .nmos_node.json
```

- `--href` 必须是**第二台的 IP + IS-05 端口**，这样在本机或第二台打开 admin 做「连接」时，请求会发到第二台。
- 若 9090 被占用，可改用 9091：`--href http://192.168.1.110:9091/`，下面 IS-05 也改为 `IS05_PORT=9091`。
- 该脚本建议**常驻或后台运行**，维持心跳。

### 3.3 第二台运行 IS-05 服务

在**第二台**另开终端：

```bash
cd /path/to/GPT_mtl_encode_sdk
python3 routing/is05_server/app.py
```

默认监听 9090；若上面用了 9091，则执行：`IS05_PORT=9091 python3 routing/is05_server/app.py`。  
第二台防火墙需放行 9090（或 9091），以便本机浏览器访问。

### 3.4 第二台实际收流（两种方式任选其一）

**方式 A：直接运行 st2110_record（与本机发送参数一致）**

在**第二台**先启动接收，再在本机启动上面的 st2110_send：

```bash
cd /path/to/GPT_mtl_encode_sdk/build

./st2110_record --ip 239.0.0.1 --video-port 5004 --audio-port 0 \
  --width 1920 --height 1080 --max-frames 3000 recv.mp4 \
  --port kernel:enp6s0 --sip 192.168.1.110 --no-ptp
```

- `--ip` / `--video-port` 必须与本机 st2110_send 一致（239.0.0.1、5004）。
- `--port kernel:enp6s0`、`--sip 192.168.1.110` 为第二台收流网卡与 IP。
- 收完后得到 **recv.mp4**（H.264 + 容器）。

**方式 B：在 admin 里点「连接」驱动收流（IS-05 + daemon）**

1. 第二台在运行注册脚本与 IS-05 服务之外，再运行收流 daemon：
   ```bash
   cd /path/to/GPT_mtl_encode_sdk/build
   ./is05_receiver_daemon
   ```
   daemon 会读取 `connection_state.json`（由 IS-05 服务在 admin 执行 Connect 时写入），按其中的组播地址/端口创建收流并写 MP4。

2. 在 admin 中对第二台的「Video Receiver (ST2110)」执行 **Connect**，选择的 Sender 对应的流参数（组播 IP/端口）需与本机 **st2110_send** 一致（239.0.0.1、5004）。若使用 Easy-NMOS 自带的虚拟 Sender，需在其配置或 SDP 中指定相同组播与端口；若使用本机注册的自研 Sender，当前示例不会自动下发 transport，可手动在 Connect 界面填写 destination 239.0.0.1、port 5004（视 Controller 界面而定）。

3. 本机仍按 §2.4 启动 **st2110_send**（--sip 192.168.1.100）发送 build/yuv 到 239.0.0.1:5004，第二台 daemon 即可收到并录成文件。

通常为简化验证，**先按方式 A** 确认本机发、第二台收与 build/yuv 播放正常，再在 admin 中做发现、连接等操作。

## 四、在 Easy-NMOS 中查看与操作

### 4.1 打开路由管理界面

在本机或第二台浏览器打开：**http://192.168.1.100/admin**。

### 4.2 查看节点与资源

- **Nodes**：应能看到本机注册的「发送节点」、第二台注册的「接收节点」。
- **Senders**：应能看到本机的「Video Sender (ST2110)」。
- **Receivers**：应能看到第二台的「Video Receiver (ST2110)」。

若未看到，检查：本机/第二台注册脚本是否在跑、REGISTRY_URL 是否为 192.168.1.100、心跳是否正常。

### 4.3 执行连接（路由）

- 在 **Receivers** 中找到第二台的「Video Receiver (ST2110)」，执行 **Connect**（或 Staged/Active 相关操作）。
- 在界面中选择要连接的 **Sender**（如本机的「Video Sender (ST2110)」或 Easy-NMOS 自带的虚拟 Sender）。
- 若 Controller 要求填写传输参数，组播地址填 **239.0.0.1**，视频端口填 **5004**，与本机 st2110_send 一致。
- 确认/激活后，IS-05 请求会发到第二台（192.168.1.110:9090），第二台 IS-05 服务会写 connection_state.json；若第二台运行了 is05_receiver_daemon，则会按该状态收流并写 MP4。

### 4.4 监控与管理

在 admin 中可查看各 Node/Sender/Receiver 状态、当前连接关系，可断开或改连，实现集中监控与统一管理。

## 五、操作顺序速查（本机发 build/yuv、第二台收）

| 步骤 | 位置 | 操作 |
|------|------|------|
| 1 | 本机 | 确认 Easy-NMOS 运行，浏览器可打开 http://192.168.1.100/admin |
| 2 | 本机 | 运行注册脚本（--mode sender），保持心跳；admin 的 Senders 中可见本机发送端 |
| 3 | 本机 | 直连网卡配 IP（若尚未配置），如 192.168.1.100/24 on enp4s0 |
| 4 | 第二台 | 网卡配 IP，如 192.168.1.110/24 on enp6s0；能 ping 通 192.168.1.100 |
| 5 | 第二台 | 运行注册脚本（--mode receiver，--href http://192.168.1.110:9090/），--save-config .nmos_node.json，保持心跳 |
| 6 | 第二台 | 运行 IS-05 服务：python3 routing/is05_server/app.py；放行 9090 |
| 7 | 第二台 | 先启动收流：st2110_record --ip 239.0.0.1 --video-port 5004 ... --port kernel:enp6s0 --sip 192.168.1.110 recv.mp4 |
| 8 | 本机 | 再启动发流：st2110_send --url yuv420p10le_1080p.yuv --ip 239.0.0.1 --video-port 5004 ... --port kernel:enp4s0 --sip 192.168.1.100 |
| 9 | 本机或第二台 | 打开 http://192.168.1.100/admin，在 Nodes/Senders/Receivers 中查看本机发送端与第二台接收端，可对 Receiver 执行 Connect 等操作 |

## 六、参数对照（本机发送与第二台接收须一致）

| 参数 | 本机 st2110_send | 第二台 st2110_record |
|------|------------------|----------------------|
| 组播 IP | --ip 239.0.0.1 | --ip 239.0.0.1 |
| 视频端口 | --video-port 5004 | --video-port 5004 |
| 分辨率 | --width 1920 --height 1080 | --width 1920 --height 1080 |
| 网卡 | --port kernel:enp4s0 | --port kernel:enp6s0 |
| 本机 IP | --sip 192.168.1.100 | --sip 192.168.1.110 |

YUV 文件：本机 build 目录下示例为 **yuv420p10le_1080p.yuv**（1920×1080），发送端 --url 指向该文件即可；接收端得到 **recv.mp4**。

## 七、常见问题

- **admin 里看不到本机 Sender 或第二台 Receiver**：确认对应侧注册脚本在运行、REGISTRY_URL=http://192.168.1.100、本机 80 端口可达。
- **第二台 Receiver 无法 Connect/Staged/Active**：确认 --href 为 http://192.168.1.110:9090/、第二台已运行 IS-05 服务、第二台防火墙放行 9090。
- **第二台收不到流**：确认两机 --ip/--video-port 一致、网卡 IP 同网段、先启接收再启发送；必要时用 tcpdump 或 Wireshark 在 239.0.0.1:5004 上抓包确认有组播。
