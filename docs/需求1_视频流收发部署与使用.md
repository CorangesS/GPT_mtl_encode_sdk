# 需求1：ST2110 视频流收发 — 部署与使用

对应 **需求.md**：视频流发送和接收、PTPv2、SDP 解析。本文说明**本地回环测试**、**双机 Kernel 模式**、**双机 DPDK 模式**的配置与流程，以及**网卡配置与查看命令**，并说明**网卡被 DPDK 绑定后如何恢复为 Kernel 使用**等意外情况处理。

---

## 一、网卡配置与查看命令（必读）

### 1.1 查看网卡与接口名

```bash
ip link show
# 或
ls /sys/class/net/
```

确认直连使用的网卡接口名（如 `enp4s0`、`enp6s0`、`eth0`）。

### 1.2 查看网卡 IP 与 PCI 地址（BDF）

```bash
# 查看各网卡 IP
ip -4 addr show

# 查看网卡与 PCI BDF 对应关系
lshw -c network -businfo
```

输出示例：`pci@0000:04:00.0  enp4s0` 表示接口 `enp4s0` 对应 BDF `0000:04:00.0`。

**由接口名查 BDF**：

```bash
# 将 enp4s0 换成实际接口名
ethtool -i enp4s0
# 输出中有 bus-info: 0000:04:00.0
```

**由 BDF 查接口名**：

```bash
ls /sys/bus/pci/devices/0000:04:00.0/net/
# 输出为接口名，如 enp4s0
```

### 1.3 查看网卡当前驱动（Kernel 还是 DPDK）

```bash
# 查看所有 PCI 网卡及当前驱动、可选用驱动
sudo $MTL_ROOT/dpdk/usertools/dpdk-devbind.py --status
# 或（若 DPDK 在 PATH）
sudo dpdk-devbind.py --status
```

输出中：
- **drv=vfio-pci** 或 **drv=igc_uio** 等：该网卡被 **DPDK 占用**，内核无法使用（不能 ping、不能 `--port kernel:xxx`）。
- **drv=igc** / **drv=e1000e** 等：该网卡由 **内核驱动** 管理，可用 `--port kernel:enp4s0`。

### 1.4 网卡临时配置 IP（Kernel 模式前）

```bash
# 发送端（例：192.168.10.1）
sudo ip addr add 192.168.10.1/24 dev enp4s0

# 接收端（例：192.168.10.2）
sudo ip addr add 192.168.10.2/24 dev enp6s0
```

验证：`ip -4 addr show enp4s0` 应看到对应 `inet`。

### 1.5 固定 IP（重启不丢失）

使用项目提供的 Netplan 配置：

- 发送端：`sudo cp docs/netplan/99-st2110-sender-enp4s0.yaml /etc/netplan/`（网卡名若不同需改 yaml 内接口）
- 接收端：`sudo cp docs/netplan/99-st2110-receiver-enp6s0.yaml /etc/netplan/`
- 然后：`sudo netplan apply`

详见 [netplan/README.md](netplan/README.md)。

### 1.6 意外情况：网卡已绑 DPDK，但需要用 Kernel 发送/接收

若之前把某网卡绑给了 DPDK（vfio-pci），该网卡**不再被内核使用**，无法 `--port kernel:enp4s0`。需要先**解绑 DPDK、绑回内核驱动**：

1. **确认该网卡未被 MTL 进程占用**（关闭 st2110_send、st2110_record 等）。
2. **绑回内核驱动**（以 Intel I226/IGC 为例，BDF 换成实际值）：
   ```bash
   # Intel I225/I226 常用内核驱动为 igc
   sudo dpdk-devbind.py --bind=igc 0000:04:00.0
   ```
   其他芯片：用 `dpdk-devbind.py --status` 看该网卡 **unused** 时的驱动名（如 `e1000e`、`ixgbe`），将 `igc` 换成该驱动名。
3. **确认**：`ip link show` 中应再次出现该接口（如 enp4s0）；然后配置 IP（见 1.4 或 1.5），即可用 `--port kernel:enp4s0`。

**反之**：若要用 DPDK 模式，需将该网卡再绑回 vfio-pci（见下文「双机 DPDK 模式」），且绑定前需先 `sudo ip link set enp4s0 down`。

## 二、本地回环测试流程

本机两进程，通过回环口收发，无需物理网卡。

1. **进入构建目录**  
   `cd build`（或项目根目录，下同）。

2. **先启动接收端**（终端 1）：
   ```bash
   ./st2110_record --ip 239.0.0.1 --video-port 5004 --audio-port 0 --max-frames 600 recv.mp4 --port kernel:lo
   ```

3. **再启动发送端**（终端 2）：
   ```bash
   ./st2110_send --ip 239.0.0.1 --video-port 5004 --audio-port 0 --duration 10 --port kernel:lo
   ```
   或从 YUV 文件发送：
   ```bash
   ./st2110_send --url build/yuv420p10le_1080p.yuv --width 1920 --height 1080 --duration 30 --audio-port 0 --ip 239.0.0.1 --video-port 5004 --port kernel:lo
   ```

回环下 PTP 通常不可用，程序会自动回退到内部时间戳；也可显式加 `--no-ptp`。

## 三、双机 Kernel 模式

两机通过**内核协议栈**（Socket）收发，需两机网卡 IP 同网段、组播互通。

### 3.1 前置条件

- 两机直连或同网段；按 **§1.4 或 §1.5** 为直连网卡配置 IP（如发送端 192.168.10.1，接收端 192.168.10.2）。
- 连通性：`ping 192.168.10.2`（在发送端）、`ping 192.168.10.1`（在接收端）应通。

### 3.2 命令示例

假设发送端网卡 `enp4s0`、IP `192.168.10.1`，接收端网卡 `enp6s0`、IP `192.168.10.2`。

**接收端（B 机）先启动**：

```bash
cd build
./st2110_record --ip 239.0.0.1 --video-port 5004 --audio-port 0 \
  --width 1920 --height 1080 --max-frames 1800 recv.mp4 \
  --port kernel:enp6s0 --sip 192.168.10.2 --no-ptp
```

**发送端（A 机）后启动**：

```bash
cd build
./st2110_send --url yuv420p10le_1080p.yuv --width 1920 --height 1080 \
  --duration 30 --audio-port 0 --ip 239.0.0.1 --video-port 5004 \
  --port kernel:enp4s0 --sip 192.168.10.1 --no-ptp
```

两机 `--ip`、`--video-port`、`--audio-port`、分辨率须一致；`--port` 为各自网卡接口名，`--sip` 为各自该网卡 IP。

## 四、双机 DPDK 模式

需在两机完成 **IOMMU、大页、VFIO、网卡绑定** 后，使用网卡 **BDF** 作为 `--port`，获得更高性能。

### 4.1 系统与权限配置（两机均做）

- **IOMMU**：BIOS 开启 VT-d；内核参数加 `intel_iommu=on iommu=pt`，`sudo update-grub` 后重启。
- **大页**：`sudo sysctl -w vm.nr_hugepages=2048`（或更大/1G 大页，见 MTL 文档）。
- **VFIO**：`sudo usermod -aG vfio $USER`，加载 `vfio-pci`，配置 udev（见 MTL/DPDK 文档）；绑定前 `sudo modprobe vfio-pci`。

### 4.2 网卡绑定到 DPDK（两机直连网卡各做一次）

1. **查看 BDF**：`lshw -c network -businfo` 或 `ethtool -i <接口名>`（§1.2）。
2. **先 down 网卡**（否则绑定会报错）：
   ```bash
   sudo ip link set enp4s0 down
   ```
3. **绑定 vfio-pci**（BDF 换成实际值）：
   ```bash
   sudo $MTL_ROOT/dpdk/usertools/dpdk-devbind.py --bind=vfio-pci 0000:04:00.0
   ```
4. **确认**：`ls /dev/vfio/` 有对应设备；`dpdk-devbind.py --status` 中该网卡为 vfio-pci。

绑定后该网卡不再被内核使用，**不能**再 `--port kernel:enp4s0`；若需改回 Kernel 模式，按 **§1.6** 解绑并绑回内核驱动。

### 4.3 双机 DPDK 命令示例

发送端 BDF `0000:04:00.0`、IP `192.168.10.1`；接收端 BDF `0000:06:00.0`、IP `192.168.10.2`。组播 `239.0.0.1`，视频端口 `5004`。

**接收端（B 机）先启动**：

```bash
cd build
./st2110_record --ip 239.0.0.1 --video-port 5004 --audio-port 0 \
  --width 1920 --height 1080 --max-frames 1800 recv.mp4 \
  --port 0000:06:00.0 --sip 192.168.10.2 --no-ptp
```

**发送端（A 机）后启动**：

```bash
cd build
./st2110_send --url yuv420p10le_1080p.yuv --width 1920 --height 1080 \
  --duration 30 --audio-port 0 --ip 239.0.0.1 --video-port 5004 \
  --port 0000:04:00.0 --sip 192.168.10.1 --no-ptp
```

若遇 build timeout 或 put 失败，可加 `--lcores 0-3 --tasklets 16`（及可选 `--put-retry`、`--prefill-frames`），见主 README 或 MTL 文档。

## 五、模式与参数对照

| 项目 | 本地回环 | 双机 Kernel | 双机 DPDK |
|------|-----------|-------------|-----------|
| `--port` | `kernel:lo` | `kernel:enp4s0` 等 | `0000:04:00.0`（BDF） |
| `--sip` | 可省略或 127.0.0.1 | 各机直连网卡 IP | 各机直连网卡 IP |
| 网卡状态 | 无需配置 | 内核驱动 + 已配 IP | 已绑 vfio-pci，内核不可用 |
| 查看/恢复 | - | §1.2～§1.5 | §1.3、§1.6、§4.2 |

## 六、常见故障

- **SIOCGIFADDR / get ip fail**：网卡名错误或该网卡未配置 IP；用 `ip link`、`ip -4 addr` 确认，按 §1.4 配 IP。
- **routing table indicates that interface is active**：绑定 DPDK 前需先 `ip link set <接口> down`。
- **收不到包**：两机 `--ip`/`--video-port` 一致、`--sip` 为直连网口 IP、`--port` 为实际使用的接口或 BDF；双机 DPDK 时确认网卡已绑 vfio。
- **网卡想用 Kernel 但被 DPDK 占用**：按 **§1.6** 解绑并绑回内核驱动（如 igc）。
